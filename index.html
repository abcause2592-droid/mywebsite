How I Built This
Design System Skill
01
Found an inspiration website
Found a site that looked exactly how I wanted mine to feel: serious, editorial, the kind of thing that belongs somewhere expensive. That was the only reference I needed.
02
Used Claude in Chrome to extract the design DNA
Pointed Claude at the inspiration site and asked it to document the full design system: color palette with exact hex codes, typeface names and weights, spacing values, layout principles, and brand vibe. Everything extracted from the live site. No guessing.
03
Used /skill-creator to turn it into a reusable skill
Packaged the extracted design system into a structured SKILL.md file using the skill creator. The file becomes a set of instructions Claude reads automatically on every invocation: colors, type specs, layout rules, and brand vibe all baked in.
04
Now it works everywhere: /abc-design
Every build that invokes the skill automatically inherits the full design system. No re-explaining the brand, no drift between outputs — it just works.

Layout Principles
  • Full-bleed heroes — edge-to-edge sections, centered max-width container for content
  • 100px vertical rhythm — generous section spacing; crowding breaks the brand
  • Border-radius: 2px — near-square corners on all buttons and UI elements, never rounded
  • Copper rules — dividers and accent borders use Brand Copper, never generic gray
  • White cards on cream — surfaces sit slightly elevated above the background field
Brand Vibe
Authoritative Editorial Measured Institutional Prestige

Every decision should pass this filter: does it belong in The Economist, a Goldman Sachs annual report, or a museum exhibition? If yes, proceed. If it feels startup-y or trendy, pull back.

How I Built This
Portfolio Website
01
Started with a simple scaffolded HTML page
Asked Claude to build a single-file portfolio with the sections I needed: Hero, About, Work, AI Projects grid, and Contact. No design system yet, just clean structure and placeholder content to establish the layout.
02
Applied my design system skill to restyle it
Invoked /abc-design and had Claude restyle everything — out with the dark theme, in with the full palette, type stack, and spacing system.
03
Claude interviewed me to fill in real content
Rather than writing it myself, Claude asked me questions — name, role, what each section should actually say. It wrote the copy and placed it. I just answered.
04
Built out project cards as I completed each sprint
Each AI Builder sprint produces a real project. As I finished each one, I came back and built out its card, adding the How I Built This and Documentation popups with actual content, not placeholder text.
Documentation
Portfolio Website
Techniques Used
  • Single-file HTML — entire site in one self-contained .html file, no build tools or dependencies
  • CSS custom properties — brand tokens defined once at :root, applied consistently throughout
  • Google Fonts only — Cormorant Garamond, Crimson Pro, and Open Sans via a single <link> tag
  • IntersectionObserver — scroll-triggered fade-in for cards, no animation library required
  • Pre-baked modal content — each popup's HTML lives in the file and is swapped in via JS, no server needed
  • /abc-design skill — enforced brand consistency across every element without manual checking
  • AI-written copy — About section and project descriptions written by Claude from a short interview
Key Prompts — Build Your Own
1. Scaffold the structure
"Build me a single-page portfolio HTML file. Interview me about any info you don't have. Sections: Hero with my name and title, About (conversational), Work (2–3 focus areas), AI Projects grid with cards that open popups, and a Contact footer. Google Fonts only."
2. Apply your design system
"Use /abc-design to restyle my portfolio. Google Fonts are the only allowed external resource. Replace the current theme with my brand system — colors, typography, spacing rules, and button styles."
3. Build out a project card
"Build out the [Project Name] card in my portfolio. 'How I Built This' popup: [describe your steps]. 'Documentation' popup: [describe what to show]. Keep all other cards as Coming Soon with empty popups."
4. Create your design system skill first
"Use /skill-creator to turn this design system into a skill I can reuse everywhere. Call it [name]. No evals needed — just write the skill and package it as a .skill file I can install in Cowork and Chat."
