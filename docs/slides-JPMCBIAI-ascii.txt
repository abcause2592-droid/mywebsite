    That's where AI has the most leverage.

+------------------------------------------------------------------------+
|  FINDING 1                                                   [04 / 10] |
|                                                                        |
|  COWORK FILLS THE "BETWEEN THE TOOLS" LAYER                           |
|  ───────────────────────────────────────────                           |
|                                                                        |
|   What it does:                                                        |
|   ·  Generates reports from files — no terminal, no code              |
|   ·  Formats, summarizes, packages output for stakeholders            |
|   ·  Connects to Calendar, Gmail, Slack, Sigma out of the box         |
|                                                                        |
|   Who it fits:                                                         |
|   Non-technical BI team members who own the last mile of a report —   |
|   the formatting, packaging, and sending.                              |
|                                                                        |
+------------------------------------------------------------------------+

  Visual: Before/After workflow. Left: analyst → manual formatting →
          email. Right: analyst → Cowork → stakeholder. One step removed.

  Speaker notes:
    Cowork doesn't replace Sigma or Databricks. It handles the work
    that happens after the data is pulled — the part that costs
    30–60 min per request and requires no SQL to solve.




+------------------------------------------------------------------------+
|  FINDING 2                                                   [05 / 10] |
|                                                                        |
|  CLAUDE CODE CONNECTS TO DATABRICKS                                    |
|  ───────────────────────────────────────────                           |
|                                                                        |
|   ·  Official Databricks MCP server — available now                   |
|   ·  Uses a read-only Service Principal — no write risk               |
|   ·  Lets analysts query and prototype in natural language             |
|   ·  Pipeline work stays in version control                            |
|                                                                        |
|   Open question:                                                       |
|   Does IT allow MCP connections from the internal network?            |
|                                                                        |
|   This answer changes the entire path forward.                        |
|                                                                        |
+------------------------------------------------------------------------+

  Visual: Simple architecture diagram: Claude Code ↔ MCP ↔ Databricks.
          Red flag icon on "IT access" node.

  Speaker notes:
    This is the power-user path. Claude Code is a terminal tool —
    it needs some technical comfort. Best fit: senior analysts or the
    internal pilot team. IT access is the gating question before
    anything else on this slide matters.




+------------------------------------------------------------------------+
|  FINDING 3                                                   [06 / 10] |
|                                                                        |
|  SIGMA SPEAKS NATURAL LANGUAGE TOO                                     |
|  ───────────────────────────────────────────                           |
|                                                                        |
|   ·  Sigma has its own MCP server                                      |
|   ·  Inherits existing permissions — nothing new provisioned          |
|   ·  Enables natural language queries against live Sigma data         |
|                                                                        |
|   The workflow:                                                        |
|   Ask Claude → Claude queries Sigma → Returns formatted result        |
|                                                                        |
|   No new data access. No IT exception.                                |
|   Runs entirely on what the team already has.                         |
|                                                                        |
+------------------------------------------------------------------------+

  Visual: Linear flow: User prompt → Claude → Sigma MCP → Data result.
          "Existing permissions" badge on the Sigma node.

  Speaker notes:
    This is the lowest-friction path to AI-assisted BI. No pipeline
    changes, no new credentials. If the team already lives in Sigma,
    this is the fastest first win.




+------------------------------------------------------------------------+
|  FINDING 4                                                   [07 / 10] |
|                                                                        |
|  EARLY CAREERS IS THE CLEAREST STARTING POINT                         |
|  ───────────────────────────────────────────                           |
|                                                                        |
|   ·  High-volume, repeatable cohort workflows                         |
|   ·  Structured data: applications, stages, outcomes                  |
|   ·  44% of Early Careers BI pain tied to Gem limitations             |
|      (Gem, 2024)                                                       |
|                                                                        |
|   Before committing:                                                   |
|   Talk to one Early Careers PM.                                       |
|   Confirm the Gem pain point is real — not assumed.                   |
|                                                                        |
+------------------------------------------------------------------------+

  Visual: Funnel diagram of Early Careers workflow stages.
          Gem labeled as "bottleneck" with 44% stat callout.

  Speaker notes:
    "Clearest" doesn't mean easiest — it means the problem is the most
    defined. Cohort reporting is repetitive, well-scoped, and produces
    measurable output. Good conditions for a pilot.
